01 大佬官方

OpenAI 发布 GPT-Red：自动化红队系统对抗提示注入

"Introducing GPT-Red — An internal automated red teamer on a mission to find our models’ prompt injection vulnerabilities at scale, helping us build stronger defenses before wider deployment."

实用点

对 agent 开发者而言，GPT-Red 展示了如何用 AI 自动化安全测试。其核心思路——用当前模型提升下一代模型的鲁棒性——可直接借鉴到自己的 agent 安全评估流程中。GPT-5.6 Sol 经此训练后，提示注入失败率比四个月前的最佳生产模型降低 6 倍。

Kimi-K3 登顶 Frontend Code Arena，76% 胜率碾压 Claude Fable 5 和 GPT-5.6 Sol

"Kimi-K3 just topped the Frontend Code Arena with a 76% pairwise win rate. When its output was compared head-to-head against other models on the same task, it was picked as the better output 76% of the time on average. For reference: Claude Fable 5 (63%), GPT-5.6 Sol (58%)."

06 关注动态

OpenAI 调查 GPT-5.6 意外删除文件事件，发布缓解措施

"We’ve investigated a handful of reports where GPT-5.6 unexpectedly deleted files. What we have found is that this most commonly occurs when: Full access mode is enabled and codex is run without sandboxing protections... The model makes an honest mistake and mistakenly deletes $HOME instead."

实用点

这是 agent 安全性的经典案例。OpenAI 的缓解措施（更新开发者消息、引导用户使用更安全的权限模式、增加 harness 防护）对所有构建 agent 的团队都有参考价值。核心教训：永远不要在生产环境中让 agent 拥有无沙箱的完全文件访问权限。
