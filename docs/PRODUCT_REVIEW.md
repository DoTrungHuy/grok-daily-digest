## 1. 核心发现：邮箱看不到完整内容（P0 致命）

### 1.1 现象

- 邮件 / 归档里只有 **HOOK + META + ITEM1 半截**，在「实用 …」处断开  
- META 写「条目数：8」，正文里 **实际只有 ITEM1**  
- 用户无法在邮箱或 `digests/*.md` 读完今日 digest  

### 1.2 根因（已用 Gmail API 全量 MIME 验证）

| 检查项 | 结果 |
|--------|------|
| 邮件 MIME | 仅 `text/html`，约 15KB |
| HTML 内可见正文 | **Grok 官方模板故意截断** |
| 完整 8 条是否在邮件里 | **否**（全文检索仅 1 个 ITEM） |
| 完整内容在哪 | 按钮 **Continue reading** → `https://grok.com/chat/<id>` |
| 今日样例全文链接 | `https://grok.com/chat/ad76e012-dafd-4721-b513-4ae9b14df03d` |
| 是否本仓库解析 bug 丢了 7 条 | **基本不是**；邮件源就没有 7 条 |

**结论：**

> 当前管道「Grok Tasks → Email → 归档」拿到的是 **预告片**，不是 **正片**。  
> 在修好「全文获取」之前，任何 Pages 上线都是把残缺内容做得好看，**产品不可用**。

### 1.3 产品责任（自我批评）

此前验收停在「能读到邮件、Actions 绿」，**没有验收「读者能否读完 8 条」**。  
这是产品经理失职，不是用户不会用。

## 3. 解决方案选项（请你选型）

### 针对 P0-1：如何拿到「完整正文」

#### 方案 A — 缩短输出，争取整封进邮件（实验，成本最低）

**做法：**  
改 prompt：强制「全文不超过 N 字 / 只 5 条 / 每条 2 行」，禁止写「8 条却很长」。  
连续测 2～3 天，看 Grok 邮件是否仍截断。

| 优点 | 缺点 |
|------|------|
| 不改架构、免费 | **可能仍被模板截断**（官方像是固定「预览+Continue」） |
| 立刻可试 | 内容变短，深度下降 |

**建议：** 先做 1 次实验，**不作为唯一依赖**。

---

#### 方案 B — 归档「预览 + 全文链接」，人工一点开（诚实 MVP）

**做法：**  
解析 HTML 里的 `https://grok.com/chat/...`，写入 digest：

- 今日预览（邮件里有的）  
- **「阅读全文」→ grok.com 链接**（登录后可见完整 8 条）

| 优点 | 缺点 |
|------|------|
| 实现快、稳定 | **仍要打开 Grok**，与「完全被动」冲突 |
| 至少不假装有全文 | 无法在 Pages 上展示完整正文 |
| 适合个人过渡 | 不算真正「上线内容产品」 |

---

#### 方案 C — 自动打开 grok.com/chat 抓取全文（完整自动化，工程重）

**做法：**  
邮件解析出 chat URL → 用 **已登录会话**（Playwright + 本地/加密 cookie，或定期导出 storage state）抓取页面正文 → 写入 digests / Pages。

| 优点 | 缺点 |
|------|------|
| 最接近「全自动完整归档」 | 脆弱：登录过期、页面改版、Cloudflare |
| 可上线展示全文 | 维护成本高；可能触碰 ToS 灰区 |
| 仍可不花 xAI token | 需要你本机或自建 runner 保活会话（纯 GitHub Actions 很难长期登录 Grok） |

**建议：** 若坚持「完整 + 自动 + 免费」，这是主候选，但要接受 **会话运维**。

---

#### 方案 D — 换内容源头（架构升级）

| 子方案 | 说明 |
|--------|------|
| D1 | **xAI API** 每日跑同一 prompt（你曾否决费用/额度） |
| D2 | 浏览器自动化直接跑 Grok（类似 C，不依赖邮件） |
| D3 | 放弃 Grok 生成，改 RSS/X 列表 + 本地规则摘要（质量/风格变） |

| 优点 | 缺点 |
|------|------|
| D1 最稳、最像产品 | 与「永不 API」冲突；有成本 |
| D2 免费 | 同 C，脆弱 |
| D3 可控 | 失去 Grok 策展质量 |

---

#### 方案 E — 产品定位降级（不推荐作最终态）

邮件只做「今日已生成」通知，完整内容永远在 Grok App 看。  
本仓库只存链接日志。

→ **不满足你「部署上线」的最终目的。**

## 4. 上线 MVP 重新定义（通过才算「部署成功」）

必须同时满足：

1. **全文完整**：ITEM 条数与 META 一致，无「实用…」腰斩  
2. **公开 URL**：GitHub Pages（或等价）打开即今日  
3. **历史列表**：至少近 7 日可点  
4. **时间对齐**：`gork-daily` 15:00 后约 30～60 分钟内站点更新  
5. **连续 3 天**无需人工复制粘贴正文  

当前：**0/5 完全满足**（邮件日更有，但是残缺；Pages 无；时间不对齐）。
